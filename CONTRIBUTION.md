# Contributing to Betalift

Thank you for your interest in contributing to Betalift! Your help is greatly appreciated. Please follow these guidelines to ensure a smooth contribution process.

## How to Contribute

1. **Fork the repository** and create your branch from `main`.
2. **Clone your fork** and set up the project locally.
3. **Create a descriptive branch name** (e.g., `feature/add-login`, `fix/feedback-bug`).
4. **Make your changes** with clear, concise commits.
5. **Test your changes** before submitting a pull request.
6. **Push to your fork** and submit a pull request (PR) to the `main` branch.

## Code Style

- Use consistent formatting and follow the existing code style.
- For TypeScript/JavaScript, use [Prettier](https://prettier.io/) and [ESLint](https://eslint.org/) (see `mobile/` and `server/` for configs).
- Write clear, descriptive commit messages.
- Add comments where necessary for clarity.

## Pull Request Process

- Ensure your PR addresses a single purpose.
- Reference related issues in your PR description (e.g., `Closes #123`).
- Include screenshots or test evidence for UI changes.
- Wait for a project maintainer to review your PR.
- Respond to feedback and make requested changes promptly.

## Reporting Issues

- Use the [ISSUES.md](ISSUES.md) guidelines for reporting bugs or requesting features.
- Provide as much detail as possible (steps to reproduce, expected/actual behavior, screenshots, etc.).

## Community Standards

- Be respectful and inclusive.
- Follow the [Code of Conduct](CODE_OF_CONDUCT.md) if available.

Thank you for helping make Betalift better!
