# Security Policy

Thank you for helping keep Betalift secure! Please follow these guidelines to report vulnerabilities and follow security best practices.

## Reporting a Vulnerability

- **Do not open public issues for security vulnerabilities.**
- Email the maintainers at [security@betalift.com](mailto:security@betalift.com) with details.
- Include a detailed description, steps to reproduce, and potential impact.
- We will acknowledge your report within 3 business days and work to resolve the issue promptly.

## Supported Versions

- Only the latest major version of Betalift is supported for security updates.

## Security Best Practices

- Keep your dependencies up to date.
- Do not share sensitive information in issues or pull requests.
- Use strong, unique passwords for your accounts.
- Enable two-factor authentication where possible.

## Responsible Disclosure

- Please give us time to investigate and address vulnerabilities before disclosing them publicly.

Thank you for helping protect the Betalift community!
